Test Selection Tool Installation

This tool aim to select the tests impacted by the lastest modifications on the source code. This plugin records the changes made in the IDE and suggest a list of test that can be relaunched by the developer.

NB: This tool needs the Test Recording tool to work. Consequently, the information dedicated to tests runs in the IDE, we be also send anonymously to the server
The plugin is compatible with Eclipse (and IntelliJ in few weeks). The install processes are described below for each IDE. You can install the plugin on several IDEs on the same computer.

For Eclipse

Installation

The installation is made through this update site: http://paia.atosworldline.com/repository/.

  1. Open your Eclipse
  2. In Help, go to Install New Software...
  3. In the text input file, paste: http://paia.atosworldline.com/repository/ and press Enter
  4. Select Test Selector item and press Next then Finish. If you get an error, check your proxy.
  5. When the security warning happens, press OK
  6. Be sure that your proxy exceptions contains: "*.priv.atos.net" (Excepted some external people).
  7. Restart Eclipse
  8. Launch a test (via Maven or JUnit)
To be sure that the plugin is correctly installed, open the Error Log (Window >> Show View >> Others and search Error Log). And search for: Plugin Started: <the date> with the plugin name com.worldline.vbu.testselector and check that for there is no error related to it. On any problem, please contact me and send the Log (you can do an export).

Supported versions

Eclipse Luna (4.4) to Neon (4.6)

For IntelliJ

Installation

The installation consist in setting up this plugin and this plugin.

  1. If needed, uninstall the previously installed testrecorder plugin
    1. In your IntelliJ IDEA, Open the Settings (File >> Settings...)
    2. In Plugin list, click on Test Recorder then on Uninstall
    3. Restart your ide
  2. Download the plugin (see links above)
  3. In your IntelliJ IDEA, Open the Settings (File >> Settings...)
  4. For the two plugins do:
    1. In Plugin, click on Install plugin from disk...
    2. Locate the previously downloaded zip and click Ok
    3. Click on OK in the settings window
  5. Restart IntelliJ IDEA
  6. Launch a test (via Maven or JUnit)
To be sure that the plugin is correctly installed and is recording, open the Log file (Help >> Show Log in Explorer and open idea.log). And search for: INFO - intellij.TestRecorderActivator - Plugin Started: and for Sync memory successful . If you do not have these entries, search first in idea.log.1 and if the issue is still present please contact me and please send the Log file.

Supported versions

IntelliJ 2017.1. But it may work on older versions (not tested).

Features

Test Selection Window

For Eclipse, this window automatically opens at startup on the bottom. If the window does not open, you can do:
Windows >> Show View >> Others... >> Other >> List of test impacted
For IntelliJ, you have to open the window through View >> Tool Windows >> List of Test Impacted

This window displays automatically the tests that are selected to be relaunched and their execution status. Green stand for test passing, yellow for assertion failure, and red for error. Gray means that the tests has not been launched.
You will be able to:
  • Choose a strategy to trigger the test selection, i.e. to choose when the tests will be selected, between:
    • After 5 minutes: All the tests that have been impacted within a period of 5 minutes since the first change after the last test run session are selected
    • At each change: The impacted tests are selected immediately after each change in the application (save, file update,...)
    • On commit change: the tests will be selected when a commit is done (works if the commit is made outside the IDE)
    • Do nothing: the test selection is not triggered.
  • Choose to run automatically the impacted tests: The checkbox Auto run selected tests allows to enable/disable it
  • Reset: remove all the tests in the selection
  • Force launch tests: launch the tests that are in the list
If you encounter any issue with plugin, you can desactivate it in the preferences of the plugin (before clicking on OK, please check that the server URL is set to: http://paia.atosworldline.com/testSelection/ ).

Test Recording Tool Installation

This tool monitors all the tests (JUnit) that will be launched in the IDE: Maven or JUnit direct execution, and anonymously sent to a server.

NB: The data will only be used for research purposes!
The plugin is compatible with Eclipse and IntelliJ. The install processes are described below for each IDE. You can install the plugin on several IDEs on the same machine.

For Eclipse

Installation

  1. Open your Eclipse
  2. In Help, go to Install New Software...
  3. In the text input file, paste: http://paia.atosworldline.com/repository/ and press Enter
  4. Select Test Recorder item and press Next then Finish. If you get an error, check your proxy.
  5. When the security warning happens, press OK
  6. Be sure that your proxy exceptions contains: "*.priv.atos.net" (Except some external people).
  7. Restart Eclipse
  8. Launch a test (via Maven or JUnit)
To be sure that the plugin is correctly installed, open the Error Log (Window >> Show View >> Others and search Error Log). And search for: Plugin Started: <the date> with the plugin name com.worldline.vbu.testrecorder and search for Sync memory successful . Specific features

The plugin should be totally invisible for the developper (excepted a trace in the error log at each test launch). However, if you encounter an issue, you can disable it easily. In Window > Preferences, go to Test Recorder Settings and select Disable plugin.

Supported versions
