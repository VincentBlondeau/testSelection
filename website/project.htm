Objective

Reduce the number of tests to execute after a modification in the source code. The test selection tool will give the opportunity to developer to spend less time on testing while still discovering possible errors in your code.

Test selection approaches

Test selection creates a mapping between tests and the code they execute. After a modification is made, one uses this mapping to find the tests that exercise exactly the changed part. Tests not affected by the change are not selected and thus not run. Therefore, for a small change, only a few tests need to be re-run, while still guaranteeing that new errors will be caught.

We did some experiments to compare both approaches on Worldline projects. The results are detailed in this research paper:
«Test Case Selection In Industry: An Analysis of Some Issues» (appeared in Software Quality Journal).
